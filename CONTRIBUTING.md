# Contributing to Compass Open Library

Thank you for your interest in helping grow the UBI Global Compass library! The product ecosystem thrives when we work together to ensure terminology, metrics, and resources are accurate and up-to-date.

To ensure the quality and consistency of the library, all contributions are reviewed by the UBI Global team before being published.

We offer three straightforward ways to contribute, designed to adapt to your preferred workflow:

## Option 1: The Quick Feedback Form (Recommended for New Users)
If you have a quick suggestion, a new term you’d like to see, or have spotted a typo, you can submit it directly through our feedback form. You do not need a GitHub account for this.

👉 **[Submit feedback or a new term here]({INSERT_FORM_LINK_HERE})**

Our automation will process your suggestion and submit it for our team's review.

## Option 2: The GitHub Web Editor (For directly editing content)
If you have a GitHub account and want to propose direct edits to the documentation, you can do so entirely within your web browser. No need to use a terminal or install Git!

1. **Find the file:** Navigate to the `docs/` folder in this repository and click on the specific Markdown (`.md`) file you wish to improve.
2. **Edit:** Click the ✏️ (pencil) icon in the top right corner of the file view.
   * *GitHub will automatically create a fork of the repository for you.*
3. **Make your changes:** Edit the text directly in the browser. 
4. **Propose changes:** Scroll to the bottom, write a brief description of what you changed and why, and click **Propose changes**.
5. **Create Pull Request:** Click **Create pull request** to send your changes to the UBI Global team for review.

## Option 3: Local Clone / CLI (For Developers & AI Agents)
If you prefer traditional Git workflows or you are delegating tasks to an AI coding assistant (e.g., Antigravity, GitHub Copilot workspace), you can use the standard CLI approach:

1. **Fork** the repository and **clone** it locally:
   `git clone https://github.com/YOUR-USERNAME/ubi_compass.git`
2. **Branch** off `main`:
   `git checkout -b feature/your-term`
3. Edit the Markdown files within the `docs/` folder.
4. **Commit** and **Push** your changes to your fork.
5. Create a **Pull Request** targeting the `main` branch of `ubiglobalse/ubi_compass`.

### Contribution Guidelines
* **Tone:** Maintain an objective, professional tone. 
* **Media:** If you are suggesting new images, please note that in your Pull Request so we can handle media hosting.
* **Review:** All pull requests are reviewed by our team. We may ask for clarifications or suggest tweaks before merging your changes into the official GitBook.

We look forward to your contributions!
