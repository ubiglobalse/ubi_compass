# Readability Standards

## Goal

Make Compass pages easy to scan, trustworthy at a glance, and understandable without prior insider knowledge of incubator and accelerator terminology.

## Shared Rules

- Start every page with a concise summary in plain language.
- Use clear section headings instead of long undifferentiated paragraphs.
- Prefer short paragraphs and focused bullet lists over dense text blocks.
- Keep canonical titles in the H1 and avoid legacy slug language in visible headings.
- Preserve related links so readers can continue exploring from the current page.

## Freshness And Trust Signals

- Show a published date when the source publication provides one.
- If no reliable published date is available, include a review or migration date only in a clearly labeled source section.
- Keep source provenance visible on migrated pages so readers can judge authority.
- Do not invent publication dates when the source does not provide them.

## Glossary Pattern

Recommended order:

1. Summary
2. Definition
3. Why It Matters
4. Industry Context
5. Visual Aid or Video Explainer when useful
6. Related Terms
7. Sources

## KPI Pattern

Recommended order:

1. Summary
2. Definition
3. Interpretation
4. Why It Matters
5. Advantages or Limitations when relevant
6. Visual Aid or Video Explainer when useful
7. Related Metrics
8. Source

## Publication Pattern

Recommended order:

1. Summary
2. Publication Details
3. Highlights or Key Takeaways
4. Cover or supporting image
5. Video when useful
6. Access the report
7. Related Publications
8. Source

## Mobile Legibility Checks

- The first screenful should explain what the page is about before the reader reaches heavy detail.
- Images should support orientation and not push the summary too far down.
- Embedded media should supplement the page instead of replacing the written explanation.
- Long bullet sections should be broken up by informative headings when possible.
